      Proxmox

      概述

      Proxmox Virtual Environment（Proxmox VE）是一个开源的服务器虚拟化管理平台，基于 Debian GNU/Linux，支持 KVM 虚拟机和 LXC 容器两种虚拟化技术，提供强大的 Web 管理界面。

      核心功能：

      • 虚拟机与容器管理：支持 QEMU/KVM 全虚拟化和 LXC 轻量级容器
      • 集群管理：支持多节点集群，实现高可用和集中管理
      • 存储管理：支持 ZFS、Ceph RBD、LVM、NFS、iSCSI 等多种存储方案
      • 网络管理：内置软件定义网络（SDN）功能
      • 备份与恢复：集成备份管理器，支持增量备份和快照
      • 高可用性：支持虚拟机和服务器的自动故障转移

      典型应用场景：

      • 企业私有云和虚拟化基础设施建设
      • 开发测试环境搭建
      • 数据中心服务器整合
      • 网络安全实验室（SOC 实验室）构建

      硬件要求

      项目 最低配置 推荐配置
      CPU 64位处理器，支持 Intel VT-x / AMD-V 8 核+
      内存 4 GB 16 GB+（生产环境 32 GB+）
      硬盘 20 GB 200 GB+ SSD（系统盘） + 数据盘
      网络 千兆网卡 双千兆网卡（支持链路聚合）

      说明：生产环境建议使用独立的存储设备安装系统，虚拟机数据存放在单独的存储池中，便于重装系统时不丢失虚拟机数据。

      部署方式

      方式一：ISO 镜像安装（推荐）

      这是官方推荐的安装方式，适合大多数用户。

      1. 下载 ISO 镜像

      访问 Proxmox 官网下载页面，下载最新的 ISO 镜像文件。

      wget https://enterprise.proxmox.com/iso/proxmox-ve_8.3-1.iso

      2. 制作启动盘

      使用 Ventoy 或 Rufus 等工具将 ISO 刻录到 USB 启动盘：

      # Linux 下使用 dd 命令
sudo dd if=/path/to/proxmox-ve.iso of=/dev/sdX bs=4M status=progress && sync

      注意：将 /dev/sdX 替换为 USB 设备的实际名称。

      3. 安装流程

      1. 从 USB 启动，选择 Install Proxmox VE (Graphical) 进入图形安装界面
      2. 阅读并接受 EULA 协议
      3. 选择目标磁盘和文件系统（推荐 ext4 或 ZFS）
        • 如需使用 ZFS RAID，在 Options 中配置
        • 重要提示：ZFS 不应部署在硬件 RAID 之上，否则可能导致数据丢失
      4. 配置地区、时区和键盘布局
      5. 设置 root 密码（至少 8 位，建议使用强密码）和邮箱地址
        • 邮箱用于接收系统通知，如软件更新和 cron 任务错误信息
      6. 配置网络（静态 IP、网关、DNS）
        • 安装期间只能配置 IPv4 或 IPv6 其中之一，双栈可在安装后添加
      7. 确认配置无误后点击 Install，等待安装完成
      8. 移除安装介质并重启

      方式二：在已有 Debian 上安装

      适用于需要自定义分区布局的高级用户，仅推荐有经验的用户使用。

      1. 安装标准 Debian

      安装标准 Debian 12（Bookworm）64位，仅选择 standard 软件包集合，不要安装桌面环境。

      2. 配置 /etc/hosts 

      vim /etc/hosts
      127.0.0.1       localhost.localdomain localhost
192.168.15.77   prox4m1.proxmox.com prox4m1

::1     localhost ip6-localhost ip6-loopback
ff02::1 ip6-allnodes
ff02::2 ip6-allrouters

      验证配置：

      hostname --ip-address
# 应返回配置的 IP 地址

      注意：确保 /etc/hosts 中没有 127.0.1.1 条目，否则可能导致服务启动失败。

      3. 添加 Proxmox VE 仓库 

      echo "deb [arch=amd64] http://download.proxmox.com/debian/pve bookworm pve-no-subscription" > /etc/apt/sources.list.d/pve-install-repo.list

      添加仓库 GPG 密钥：

      wget http://download.proxmox.com/debian/proxmox-release-bookworm.gpg -O /etc/apt/trusted.gpg.d/proxmox-release-bookworm.gpg
chmod +r /etc/apt/trusted.gpg.d/proxmox-release-bookworm.gpg

      4. 更新系统并安装 Proxmox VE 

      apt update && apt full-upgrade
apt install proxmox-ve postfix open-iscsi

      安装过程中：

      • 如需配置邮件，选择 Satellite system，配置现有邮件服务器作为中继
      • 如不确定，选择 Local only

      5. 重启系统 

      reboot

      系统将自动引导至 Proxmox VE 内核。

      首次登录与初始化配置

      1. 访问 Web 管理界面

      在浏览器中打开 https://<服务器IP>:8006。

      注意：使用 https 协议，浏览器会提示证书不受信任，点击「高级」→「继续访问」即可。

      登录信息：

      • 用户名：root
      • 密码：安装时设置的密码
      • 认证方式：PAM

      2. 配置软件仓库

      Proxmox 默认启用企业版仓库（需要订阅），社区用户需切换为无订阅仓库：

      Web 界面方式：

      1. 进入 数据中心 → 节点 → 更新 → 存储库
      2. 禁用 enterprise.proxmox.com/debian/pve 企业仓库
      3. 添加 http://download.proxmox.com/debian/pve bookworm pve-no-subscription 仓库
      4. 点击 更新 → 刷新

      命令行方式：

      # 备份并替换 sources.list
cp /etc/apt/sources.list /etc/apt/sources.list.bak
cat > /etc/apt/sources.list << EOF
deb http://ftp.debian.org/debian bookworm main contrib
deb http://ftp.debian.org/debian bookworm-updates main contrib
deb http://security.debian.org bookworm-security main contrib
deb http://download.proxmox.com/debian/pve bookworm pve-no-subscription
EOF

# 删除企业仓库
rm -f /etc/apt/sources.list.d/pve-enterprise.list

# 更新
apt update && apt upgrade

      3. 合并 local 和 local-lvm（可选）

      默认情况下，系统将磁盘分为两个分区：local（存放 ISO 镜像和模板）和 local-lvm（存放虚拟机磁盘）。为充分利用空间，可将两者合并：

      # 查看当前 LVM 卷
lvs

# 删除 data 卷（确保其中无数据）
lvremove pve/data

# 将释放的空间扩展到 root 卷
lvextend -l +100%FREE -r pve/root

      然后在 Web 界面中：

      1. 进入 数据中心 → 存储
      2. 删除 local-lvm
      3. 编辑 local，勾选所有内容类型

      常用管理命令

      操作 命令
      查看集群状态 pvecm status
      创建虚拟机 qm create <VMID>
      启动虚拟机 qm start <VMID>
      停止虚拟机 qm stop <VMID>
      查看虚拟机列表 qm list
      备份虚拟机 vzdump <VMID> --mode snapshot
      查看 LVM 卷 lvs
      查看存储使用 df -h
      查看系统日志 journalctl -u pvedaemon -f
      查看集群日志 journalctl -u pve-cluster -f

      常见问题

      Q：安装后无法访问 Web 界面？

      1. 检查服务是否正常运行：
      systemctl status pveproxy
      1. 确认防火墙已放行 8006 端口：
      firewall-cmd --add-port=8006/tcp --permanent
firewall-cmd --reload
      1. 检查网络配置是否正确，确认 IP 地址与安装时配置一致。

      Q：提示"No valid subscription"订阅弹窗？

      社区版用户可通过以下命令移除订阅弹窗：

      sed -Ezi.bak "s/(Ext.Msg.show\(\{\s+title: gettext\('No valid sub)/void\(\{ \/\/\1/g" /usr/share/javascript/proxmox-widget-toolkit/proxmoxlib.js && systemctl restart pveproxy.service

      Q：虚拟机启动失败？

      1. 检查存储路径是否可访问：
      df -h
      1. 确认 KVM 虚拟化支持已启用：
      grep -E "vmx|svm" /proc/cpuinfo
      1. 检查 /etc/hosts 配置是否正确：
      hostname --ip-address

      Q：如何配置国内软件源加速更新？ 

      # Debian 系统源（163 源）
sed -i.bak "s#ftp.debian.org/debian#mirrors.163.com/debian#g" /etc/apt/sources.list

# Proxmox 源（中科大源）
wget https://mirrors.ustc.edu.cn/proxmox/debian/proxmox-release-bookworm.gpg -O /etc/apt/trusted.gpg.d/proxmox-release-bookworm.gpg
echo "deb https://mirrors.ustc.edu.cn/proxmox/debian/pve bookworm pve-no-subscription" > /etc/apt/sources.list.d/pve-no-subscription.list

apt update

      Q：忘记 root 密码怎么办？

      重启服务器，在 GRUB 菜单中选择高级选项，进入恢复模式（Recovery Mode），使用 passwd 命令重置 root 密码。
