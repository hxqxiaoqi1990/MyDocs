      Docker Compose基础

      一、Docker Compose 配置文件核心结构

      Docker Compose 配置文件（docker-compose.yml/compose.yml）基于 YAML 格式，核心分为 五大模块：services（服务，必选）、networks（网络）、volumes（卷）、configs（配置）、secrets（密钥），另支持顶层全局参数和自定义扩展字段，以下是所有支持的参数详细说明。

      二、顶层全局参数

      顶层参数作用于整个 Compose 项目，定义项目的基础配置和全局资源。

      参数 说明
      version 指定 Compose 文件格式版本（推荐 3.8+，适配 Docker 20.10+），不同版本支持的参数存在兼容性差异，3.x 版本适配 Swarm 模式，2.x 偏向单机使用
      services 定义所有容器服务（核心必选模块），下属每个子项为一个独立容器服务，包含容器运行的所有配置
      networks 定义自定义网络，用于服务间网络隔离、通信，支持多种网络驱动，需在服务中通过 networks 关联
      volumes 定义持久化数据卷，用于容器数据持久化（容器删除后数据不丢失），支持本地卷、网络卷等，需在服务中通过 volumes 挂载
      configs Docker Swarm 模式专用，定义跨服务共享的配置文件（非敏感信息），支持本地文件加载或引用外部配置
      secrets Docker Swarm 模式专用，定义敏感信息（密码、密钥等），避免明文暴露，支持本地文件加载或引用外部密钥
      x-* 自定义扩展字段，字段名必须以 x- 开头（如 x-common-env），Compose 会忽略该字段，但可通过 YAML 锚点语法复用配置

      三、Services 模块（容器服务核心参数）

      services 是 Compose 配置的核心模块，每个服务对应一个容器实例，参数覆盖容器运行的所有配置（等价于 docker run 命令的所有选项）。

      1. 基础配置参数

      参数 说明
      image 指定容器镜像（格式：镜像名:标签），与 build 二选一，优先使用已有镜像
      build 从 Dockerfile 构建镜像，不使用现有镜像，支持子参数：context（构建上下文目录，必选）、dockerfile（自定义 Dockerfile 文件名）、args（构建参数，对应 Dockerfile 的 ARG）、labels（构建时添加镜像标签）、cache_from（构建缓存来源）、cache_to（构建缓存推送目标）、shm_size（共享内存大小）
      container_name 自定义容器名称，默认自动生成（格式：项目名_服务名_序号），同一主机内容器名称唯一
      command 覆盖容器默认启动命令，支持字符串格式（如 nginx -g 'daemon off;'）或数组格式（如 ["nginx", "-g", "daemon off;"]），数组格式更安全（避免字符转义问题）
      entrypoint 覆盖容器默认入口点，优先级高于 command，支持字符串或数组格式，入口点是容器启动时执行的第一个命令
      restart 容器重启策略：no（默认，不重启）、always（容器退出时始终重启）、on-failure（容器退出状态码非 0 时重启）、unless-stopped（除非手动停止容器，否则始终重启）
      hostname 容器主机名，在容器内部可见，用于容器内程序识别主机标识
      domainname 容器域名，配合 hostname 使用，组成完整的主机域名（如 hostname: app + domainname: example.com → app.example.com）
      mac_address 容器 MAC 地址（如 02:42:ac:11:65:43），用于需要绑定 MAC 地址的场景
      privileged 是否授予容器特权模式（true/false），特权模式下容器拥有主机的所有设备访问权限
      read_only 容器文件系统是否设为只读（true/false），只读模式下容器无法修改文件系统，需通过卷挂载可写目录
      stdin_open 是否保持标准输入打开（true/false），等价于 docker run -i
      tty 是否分配伪终端（true/false），等价于 docker run -t，通常与 stdin_open 配合使用（stdin_open: true + tty: true → 等价于 docker run -it）
      stop_signal 停止容器的信号（默认 SIGTERM），支持 Unix 信号（如 SIGINT、SIGKILL）
      stop_grace_period 容器停止的宽限期（默认 10s），宽限期内容器正常退出，超时后强制终止（SIGKILL）
      healthcheck 容器健康检查配置，子参数：test（健康检查命令，如 ["CMD", "curl", "-f", "http://localhost"]）、interval（检查间隔，如 30s）、timeout（检查超时时间，如 10s）、retries（检查失败重试次数，如 3）、start_period（容器启动后延迟检查的时间，如 30s）、disable（是否禁用健康检查，true/false）
      depends_on 服务依赖关系，控制服务启动 / 停止顺序（启动时先启动依赖服务，停止时后停止当前服务），支持数组格式（如 ["db", "redis"]），仅保证启动顺序，不保证依赖服务就绪（需结合健康检查）
      deploy 部署配置（Swarm 模式核心参数，单机模式部分生效），子参数：replicas（Swarm 模式下的副本数）、resources（资源限制与预留）、restart_policy（Swarm 模式重启策略）、placement（节点部署约束）、update_config（滚动更新配置）、rollback_config（回滚配置）、labels（服务部署标签）、mode（部署模式：replicated 副本模式 /global 全局模式）
      labels 容器元数据标签，用于分类、筛选容器，支持键值对格式（如 com.example.env: production）或数组格式（如 ["com.example.team=devops", "com.example.version=1.0"]）

      2. 网络配置参数

      参数 说明
      networks 容器关联的自定义网络，需在顶层 networks 中定义，支持子参数：aliases（容器在该网络中的别名，服务间可通过别名访问）、ipv4_address（静态 IPv4 地址，需网络支持固定子网）、ipv6_address（静态 IPv6 地址）
      ports 端口映射，将容器端口暴露到主机，支持短格式（主机端口:容器端口，如 8080:80）、长格式（published 主机端口、target 容器端口、protocol 协议（tcp/udp/sctp）、mode 模式（host/ingress，Swarm 模式有效））
      expose 暴露容器内部端口（仅声明端口，不映射到主机），供同一网络内的其他服务访问，支持单个端口（如 8080）或数组格式（如 ["8080", "9090"]）
      dns 自定义容器 DNS 服务器，支持单个 DNS（如 8.8.8.8）或数组格式（如 ["8.8.8.8", "114.114.114.114"]）
      dns_search DNS 搜索域，容器解析主机名时自动拼接的域名后缀（如 ["example.com", "test.com"]），解析 app 时会尝试 app.example.com、app.test.com
      dns_opt DNS 选项，如 ["use-vc", "timeout:2"]（使用 TCP 连接 DNS 服务器、超时时间 2 秒）
      network_mode 容器网络模式，可选值：bridge（默认，桥接模式）、host（共享主机网络栈）、none（无网络）、service:>（共享其他服务的网络栈）、container:>（共享其他容器的网络栈）
      extra_hosts 添加主机名到 IP 的映射（等价于 docker run --add-host），支持数组格式（如 ["host.docker.internal:host-gateway", "db-server:192.168.1.100"]）
      hostname 容器主机名（已在基础配置中列出，此处重复标注网络相关作用）
      domainname 容器域名（已在基础配置中列出，此处重复标注网络相关作用）
      mac_address 容器 MAC 地址（已在基础配置中列出，此处重复标注网络相关作用）

      3. 存储配置参数

      参数 说明
      volumes 数据卷挂载，支持三种类型：bind（主机路径绑定）、volume（命名卷，需在顶层 volumes 定义）、tmpfs（临时文件系统），格式：短格式（源:目标:权限，如 ./data:/app/data:ro）、长格式（type 类型、source 源路径、target 目标路径、read_only 是否只读、bind 绑定选项、volume 卷选项、tmpfs 临时文件系统选项）
      tmpfs 挂载临时文件系统（容器重启后数据丢失），支持单个路径（如 /tmp）或数组格式（如 ["/tmp", "/var/run:size=100m"]），size 限制临时目录大小
      ulimits 容器资源限制，控制容器的最大进程数、文件描述符数等，支持子参数：nproc（最大进程数）、nofile（最大文件描述符数，支持 soft 软限制和 hard 硬限制）、core（核心文件大小限制）、rss（常驻内存大小限制）等
      shm_size 容器共享内存大小（如 1g、512m），等价于 docker run --shm-size
      sysctls 配置容器内核参数（如 net.ipv4.ip_forward: 1、net.core.somaxconn: 1024），支持键值对或数组格式
      cap_add 添加容器的 Linux capabilities（如 NET_ADMIN、SYS_TIME），赋予容器额外权限
      cap_drop 移除容器的 Linux capabilities（如 ALL 移除所有权限，仅保留默认最小权限）
      devices 挂载主机设备到容器（如 /dev/sda1:/dev/sda1），支持数组格式，需配合 privileged: true 使用

      4. 环境配置参数

      参数 说明
      environment 设置容器环境变量，支持数组格式（如 ["MYSQL_ROOT_PASSWORD=123456", "APP_ENV=prod"]）或字典格式（如 MYSQL_ROOT_PASSWORD: 123456），优先级高于 env_file，同名变量会覆盖 env_file 中的值
      env_file 从文件加载环境变量，支持单个文件（如 .env）或数组格式（如 [".env", "prod.env"]），文件中每行格式为 KEY=VALUE，空行和 # 开头的注释行会被忽略
      secrets Docker Swarm 模式下挂载密钥，从顶层 secrets 中引用，支持子参数：source（密钥名称）、target（容器内挂载路径）、uid（挂载文件的用户 ID）、gid（挂载文件的组 ID）、mode（挂载文件的权限，如 0400）
      configs Docker Swarm 模式下挂载配置文件，从顶层 configs 中引用，子参数与 secrets 一致，用于挂载非敏感配置
      user 容器运行用户，支持用户名（如 root）、UID（如 1000）、UID:GID（如 1000:1000），用于控制容器内程序的运行权限
      working_dir 容器工作目录（如 /app），容器启动后默认进入该目录，command 和 entrypoint 命令会在该目录下执行
      env_file 从文件加载环境变量（已说明，此处保持分类一致性）

      5. 资源限制与部署参数（补充 deploy 子参数）

      参数 说明
      cpus 简化的 CPU 限制（如 0.5、2），等价于 deploy.resources.limits.cpus，单机模式生效
      cpus_per_task Swarm 模式下每个任务（容器）的 CPU 限制，仅 mode: replicated 有效
      mem_limit 简化的内存限制（如 1g、512m），等价于 deploy.resources.limits.memory，单机模式生效
      mem_reservation 简化的内存预留（如 512m），等价于 deploy.resources.reservations.memory，保证容器最小可用内存
      deploy.resources.limits 容器最大资源限制，子参数：cpus（CPU 核心数，如 1 表示 1 核）、memory（内存大小，如 1g）、pids（最大进程数，如 50）
      deploy.resources.reservations 容器最小资源预留，子参数与 limits 一致，确保节点为容器分配足够资源
      deploy.restart_policy Swarm 模式重启策略，子参数：condition（重启条件：none/on-failure/any）、delay（重启延迟，如 5s）、max_attempts（最大重启次数，如 3）、window（判断容器稳定的时间窗口，如 120s）
      deploy.placement Swarm 节点部署约束，子参数：constraints（节点约束，如 ["node.role==worker", "node.labels.env==prod"]）、preferences（部署偏好，如 [{"spread": "node.labels.zone"}]）
      deploy.update_config Swarm 滚动更新配置，子参数：parallelism（同时更新的副本数，如 1）、delay（副本更新间隔，如 10s）、failure_action（更新失败动作：continue/rollback/pause）、monitor（更新后监控时间，如 60s）、max_failure_ratio（最大失败比例，如 0.2）、order（更新顺序：stop-first/start-first）
      deploy.rollback_config Swarm 更新失败回滚配置，子参数与 update_config 一致
      deploy.mode Swarm 部署模式：replicated（副本模式，需指定 replicas）、global（全局模式，每个节点运行一个副本）
      deploy.replicas Swarm 副本模式下的容器数量（如 3），仅 mode: replicated 有效
      deploy.labels Swarm 服务部署标签，仅作用于服务，不作用于容器

      6. 日志配置参数

      参数 说明
      logging 容器日志配置，子参数：driver（日志驱动，默认 json-file，支持 syslog、journald、fluentd、gelf、none 等）、options（驱动选项，如 json-file 的 max-size（单个日志文件大小）、max-file（日志文件保留数量）、compress（是否压缩旧日志））

      四、Networks 模块（自定义网络参数）

      networks 模块定义项目的自定义网络，用于服务间通信隔离，支持多种网络驱动，所有参数如下：

      参数 说明
      driver 网络驱动，可选值：bridge（单机默认桥接网络）、host（共享主机网络）、overlay（Swarm 跨节点网络）、macvlan（分配物理 MAC 地址，模拟物理设备）、none（无网络）、ipvlan（L2/L3 层 VLAN 网络）、第三方驱动（如 weave、calico）
      driver_opts 网络驱动选项，不同驱动支持不同选项，如 bridge 驱动的 com.docker.network.bridge.name（自定义桥接网卡名称）、overlay 驱动的 com.docker.network.driver.mtu（MTU 大小）
      ipam IP 地址管理配置，子参数：driver（IPAM 驱动，默认 default）、config（IP 配置列表，每个配置包含 subnet（子网）、gateway（网关）、ip_range（可用 IP 范围）、aux_addresses（预留 IP 地址））、options（IPAM 选项）
      external 是否使用外部已存在的网络（true/false），true 时 Compose 不创建网络，直接使用现有网络（需提前通过 docker network create 创建）
      name 自定义网络名称（默认格式：项目名_网络名），避免默认名称过长或冲突
      labels 网络元数据标签，用于分类、筛选网络
      internal 是否设置为内部网络（true/false），内部网络中的服务无法访问外部网络，仅允许内部服务通信
      attachable Swarm overlay 网络专用，是否允许非 Swarm 服务的容器连接到该网络（true/false）
      ingress Swarm 专用，是否设置为入口网络（true/false），入口网络用于 Swarm 服务的负载均衡，默认已存在 ingress 网络
      enable_ipv6 是否启用 IPv6（true/false），需配合 ipam.config 中的 IPv6 子网配置

      五、Volumes 模块（持久化卷参数）

      volumes 模块定义项目的持久化卷，用于容器数据持久化，所有参数如下：

      参数 说明
      driver 卷驱动，默认 local（本地卷），支持第三方驱动（如 nfs、glusterfs、ceph、azure_file 等）
      driver_opts 卷驱动选项，不同驱动支持不同选项，如 nfs 驱动的 type（文件系统类型，nfs）、o（挂载选项，如 addr=192.168.1.100,rw）、device（NFS 服务器共享目录，如 :/data/nfs）；local 驱动的 type（none/bind）、o（挂载选项）、device（主机路径）
      external 是否使用外部已存在的卷（true/false），true 时 Compose 不创建卷，直接使用现有卷（需提前通过 docker volume create 创建）
      name 自定义卷名称（默认格式：项目名_卷名），避免默认名称冲突
      labels 卷元数据标签，用于分类、筛选卷
      permissions 卷权限配置（部分驱动支持），如 read_only（只读）
      target 卷挂载目标路径（仅在服务 volumes 中配置，此处为模块级参数说明）
      source 卷挂载源路径（仅在服务 volumes 中配置，此处为模块级参数说明）

      六、Configs 模块（Swarm 配置文件参数）

      configs 模块仅在 Docker Swarm 模式下生效，定义跨服务共享的非敏感配置文件，所有参数如下：

      参数 说明
      file 从本地文件加载配置内容（如 file: ./app.conf），Compose 启动时将文件内容上传到 Swarm 配置存储
      external 是否使用外部已存在的配置（true/false），true 时引用 Swarm 中已创建的配置（需提前通过 docker config create 创建）
      name 自定义配置名称（默认格式：项目名_配置名），Swarm 中配置名称唯一
      template_driver 配置模板驱动（如 golang），支持使用模板动态生成配置文件（需配置文件中包含模板语法）

      七、Secrets 模块（Swarm 密钥参数）

      secrets 模块仅在 Docker Swarm 模式下生效，定义敏感信息（密码、密钥等），所有参数如下：

      参数 说明
      file 从本地文件加载密钥内容（如 file: ./db-password.txt），文件内容为敏感信息（无需 KEY=VALUE 格式），上传到 Swarm 密钥存储时会加密
      external 是否使用外部已存在的密钥（true/false），true 时引用 Swarm 中已创建的密钥（需提前通过 docker secret create 创建）
      name 自定义密钥名称（默认格式：项目名_密钥名），Swarm 中密钥名称唯一

      补充说明

      1. 版本兼容性：部分参数仅支持特定 Compose 版本，如 deploy 模块的 rollback_config 需 version: 3.7+，configs 和 secrets 需 version: 3.3+，建议优先使用 3.8 版本（最新稳定版）。

      2. 参数优先级：environment > env_file > 镜像默认环境变量；command/entrypoint 覆盖镜像默认配置；服务内 networks/volumes 引用顶层定义的资源。

      3. 官方文档参考：所有参数的最新说明可查阅 Docker 官方文档：Compose File Reference

      八、示例 

      version: '3.8'

# 自定义网络
networks:
  app-network:
    driver: bridge
    ipam:
      config:
        - subnet: 172.30.0.0/16
          gateway: 172.30.0.1

# 持久化卷
volumes:
  mysql-data:
    driver: local
    driver_opts:
      type: none
      o: bind
      device: ./mysql-data

# 服务定义
services:
  # Web 服务
  nginx:
    image: nginx:alpine
    container_name: my-nginx
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
      - ./html:/usr/share/nginx/html
    networks:
      - app-network
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "wget", "-qO-", "http://localhost"]
      interval: 30s
      timeout: 10s
      retries: 3
    depends_on:
      - php

  # 数据库服务
  mysql:
    image: mysql:8.0
    container_name: my-mysql
    environment:
      MYSQL_ROOT_PASSWORD: 123456
      MYSQL_DATABASE: app_db
    volumes:
      - mysql-data:/var/lib/mysql
    networks:
      - app-network
    deploy:
      resources:
        limits:
          cpus: "1"
          memory: 1G
        reservations:
          cpus: "0.5"
          memory: 512M
    restart: on-failure:3

# 扩展字段（复用配置）
x-common-env: &common-env
  TZ: Asia/Shanghai
  LANG: en_US.UTF-8
